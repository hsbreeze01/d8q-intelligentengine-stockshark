# Requirements Document Template

## 1. Overview
Brief description of the feature/project.

## 2. User Stories
List of user stories following the EARS format:

### 2.1 Epic: [Epic Name]
- **Story**: [As a <role>, I want <feature> so that <benefit>]
- **Acceptance Criteria**:
  - [Criteria 1]
  - [Criteria 2]
  - [Criteria 3]

## 3. Functional Requirements
Detailed functional requirements.

## 4. Non-Functional Requirements
Performance, security, usability, etc.

## 5. Data Requirements
Data sources, formats, and usage.

## 6. Constraints
Technical, business, or resource constraints.

## 7. Dependencies
External dependencies or integrations.
